# 智能灌溉APP设计

## 1. 概述

为基于STM32的智能灌溉系统开发配套手机APP，使用Qt 6 + QML实现，通过蓝牙HC-05与下位机通信。

## 2. 通信协议

- **蓝牙模块**: HC-05，波特率9600
- **数据格式**: hex二进制传输，每包4字节
- **数据字段**:
  - `bluetooth_data[0]` = 温度（整数值，已偏移）
  - `bluetooth_data[1]` = 湿度（整数值，已偏移）
  - `bluetooth_data[2]` = 土壤湿度百分比
  - `bluetooth_data[3]` = 光照强度百分比
- **指令**:
  - `STATUS` → 发送4字节hex状态数据
  - `WATER_NOW` → 触发浇水，收到ACK确认

## 3. 功能清单

| 功能 | 说明 |
|------|------|
| 蓝牙自动连接 | 启动时扫描并连接名为`HC-05`的设备 |
| 手动MAC地址连接 | 自动连接失败时，用户可在状态栏输入MAC地址连接 |
| 自动重连 | 连接断开后自动重连，最多3次 |
| 传感器数据展示 | 温度、湿度、土壤湿度、光照强度，实时刷新 |
| 折线图 | 显示4种传感器数据的历史趋势（最近30个数据点） |
| 立即刷新按钮 | 发送STATUS指令，5秒CD，按钮变灰 |
| 浇水按钮 | 发送WATER_NOW指令，5秒CD，按钮变灰 |

## 4. UI结构

```
┌─────────────────────────┐
│     智能灌溉系统         │  ← 标题栏 + 连接状态指示
├─────────────────────────┤
│  🌡️ 温度  💧 湿度  🌱 土壤  ☀️ 光照  │  ← 数据卡片
├─────────────────────────┤
│     [折线图区域]         │  ← 历史趋势
├─────────────────────────┤
│ [立即刷新]   [立即浇水]  │  ← 操作按钮
├─────────────────────────┤
│  [MAC地址输入] [连接]    │  ← 备用手动连接
│  连接状态 / 最后更新时间  │  ← 状态栏
└─────────────────────────┘
```

## 5. 技术选型

- **框架**: Qt 6.x
- **UI**: QML
- **蓝牙**: Qt Bluetooth API（QML Bluetooth类型）
- **图表**: Qt Charts 或 自定义Canvas绘制
- **数据存储**: QSettings（保存MAC地址和历史数据）
- **部署**: Android

## 6. 蓝牙连接策略

1. **启动时**: 扫描名称为`HC-05`的设备，找到后自动连接
2. **连接失败/超时**: 在状态栏显示提示，启用MAC地址手动输入框
3. **手动连接**: 用户输入MAC地址，点击连接按钮
4. **断开重连**: 自动重连最多3次，3次失败后进入手动模式
5. **MAC地址保存**: 成功连接的地址保存到本地，下次启动优先尝试

## 7. 折线图设计

- X轴: 时间（最近30个数据点，约15分钟数据，以30秒间隔计）
- Y轴: 0-100（百分比）或 0-50（温度/湿度）
- 4条曲线: 温度、湿度、土壤湿度、光照，用不同颜色区分
- 点击曲线可显示具体数值

## 8. 按钮CD机制

- 每次点击后，按钮变灰并显示倒计时（5→4→3→2→1）
- CD期间禁止重复触发
- CD结束恢复可用状态
